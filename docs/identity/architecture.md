# Identity Layer Architecture

## Overview

The identity layer (`src/platform/identity/`) provides authentication, authorization, organization management, RBAC, API keys, and session management for the CompilerAI SaaS platform. It follows DDD and SOLID principles, fully decoupled from the domain layer.

## Key Principles

1. **Interface-driven** вҖ” All repositories and auth providers are interfaces. In-memory implementations for testing; Postgres for production.
2. **Domain decoupled** вҖ” Domain code never touches JWT, API keys, or passwords directly.
3. **Multitenant** вҖ” Every resource is scoped by `organizationId`. Cross-org access is blocked by default.
4. **No secrets logged** вҖ” Passwords, tokens, and API keys are never written to logs.
5. **RBAC + custom roles** вҖ” 6 system roles + org-scoped custom roles with 18 permissions.
